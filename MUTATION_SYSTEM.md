# 錦鯉シミュレーション - 変異システム仕様書

## 概要

錦鯉シミュレーションの変異システムは、親の色構成に基づいて子の色を決定する確率的なシステムです。2026年4月6日に大規模なリファクタリングが実施され、従来の優先順位ベースのシステムから**重み付けシステム**に移行しました。

## システムの基本設計

### 重み付けシステムの仕組み

1. **条件収集フェーズ**: 親の色構成、世代数などの条件に基づいて、適用可能なすべての変異ルールを収集
2. **重み計算フェーズ**: 各ルールに設定された重み（確率）を基に総重みを計算
3. **ルール選択フェーズ**: 総重みに比例した確率で1つのルールをランダム選択
4. **適用フェーズ**: 選択されたルールの効果を適用

### 選択アルゴリズム

```javascript
// 総重みを計算
let totalWeight = applicableRules.reduce((sum, rule) => sum + rule.weight, 0);

// 重みに基づいてルールを選択
let randomValue = Math.random() * totalWeight;
let accumulated = 0;

for (let rule of applicableRules) {
    accumulated += rule.weight;
    if (randomValue < accumulated) {
        selectedRule = rule;
        break;
    }
}
```

## 通常の変異（特別な変異ルールが発動しなかった場合）

### 変異確率
- **基本変異率**: 50%
- **白黒のみの場合**: 80%（第2世代以上）

### 変異プール
変異プールからランダムに色を選択：
- 常に: 白（`#f8f9fa`）、黒（`#343a40`）
- 第2世代以降: 赤（`#ff4d4d`）
- 第3世代以降: 青（`#1aa3ff`）

### 変異効果
選択された色をベース色または模様色のどちらかに上書き（50%の確率でどちらか）

## 特別な救済措置

### 第2世代赤色強制救済
- **条件**: 第2世代で、親に赤色が含まれず、子が単色の場合
- **効果**: 子の模様色を赤色に強制変更
- **優先度**: すべての変異処理後に適用

## 確率計算の例

### 例1: 赤を含む鯉同士の交配（青色を含まない）
適用可能ルール:
- ルール4（白＋赤 -> 紅白）: 重み15（親が白も含む場合）
- ルール6（赤 -> 赤単色）: 重み10
- ルール9（赤 -> 青）: 重み3（第1世代の場合）

総重み: 15 + 10 + 3 = 28
発動確率:
- 紅白: 15/28 ≈ 53.6%
- 赤単色: 10/28 ≈ 35.7%
- 赤->青: 3/28 ≈ 10.7%

### 例2: 白黒のみの交配（第2世代）
適用可能ルール:
- ルール10（白黒のみ -> 赤柄/青柄）: 重み30
- ルール11（白単色 -> 黒単色）: 重み10（白単色の場合）

総重み: 30 + 10 = 40
発動確率:
- 赤柄: (20/40) × (30/40) = 15/40 = 37.5%
- 青柄: (10/40) × (30/40) = 7.5/40 = 18.75%
- 黒単色: 10/40 = 25%
- 通常の変異: 20% × (1 - 40/40) = 0%（特別な変異が発動するため）

## システムの特徴

### 利点
1. **公平な確率分配**: すべてのルールが重みに比例した確率で発動
2. **バランス調整が容易**: 重みを変更するだけで確率を調整可能
3. **新しいルールの追加が容易**: 既存のルールの相対的な確率を維持しながら追加可能
4. **デバッグが容易**: どのルールが発動したか追跡可能

### 設計思想
1. **単色品種は控えめに**: 単色品種の重みは10前後に設定
2. **柄入り品種を促進**: 紅白、秋翠などの柄入り品種の重みを15に設定
3. **黒色の持続性**: 黒色が失われても回復可能な仕組み
4. **世代進行**: 世代が進むほど青色が生まれやすくなる
