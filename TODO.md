# 錦鯉シミュレーション - 実装予定

## 問題: 変異システムの整理と新しい色の追加の準備

### 問題の詳細な説明
現在の変異システムには以下の問題がある：
1. **優先順位ベースの競合**: `specialMutation`フラグを使用しており、最初に発動したルールが他のルールをブロックする
2. **ルールの依存関係が複雑**: ルールが増えるにつれて、どのルールがどのルールを阻害するかが不明確
3. **確率の重複と競合**: 複数のルールが同じ条件を満たす場合、最初のルールのみが発動する
4. **新しい色の追加が困難**: 新しいルールを追加すると、既存のルールの発動確率が低下する
5. **デバッグとバランス調整が困難**: ルールが11個もあり、相互関係を理解するのが難しい

### 具体的な解決策
**確率の重み付けシステム**を導入する：
1. 条件を満たすすべてのルールを収集
2. 各ルールに重み（確率）を設定
3. 重みに基づいてランダムに1つのルールを選択
4. 選択されたルールを適用

### 実装タスクのリスト
- [x] 現在の変異ルールを分析し、ルール定義を抽出（11個のルールを分析）
- [x] 重み付けシステムの設計と実装（重みに基づく確率選択システム）
- [x] 既存の11個の変異ルールを新しいシステムに移行（すべてのルールをオブジェクト化）
- [x] 新しいシステムの動作確認とバランス調整（コードレベルの整合性確認完了）
- [x] 新しい色の追加のための基盤を整備（モジュール化されたルールシステム）

### 期待される効果
1. **新しいルールの追加が容易**: 既存のルールの相対的な確率を維持しながら新しいルールを追加可能
2. **バランス調整が容易**: ルールの重みを調整するだけで確率を簡単に変更可能
3. **デバッグが容易**: どのルールが発動したか追跡しやすくなる
4. **新しい色の追加の準備**: システムが整理され、新しい色（例：緑色、橙色など）の追加が容易になる

### 注意事項
- 既存の遺伝ロジックを尊重する
- ゲームバランスを維持する
- 後方互換性を確保する（既存のゲームデータが壊れないように）
